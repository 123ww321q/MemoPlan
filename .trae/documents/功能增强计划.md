# MemoPlan 功能增强计划

## 项目现状分析

基于对项目架构的全面分析，MemoPlan 是一个基于 Tauri + React + TypeScript 的桌面备忘录应用，已实现笔记管理、任务提取、回收站、设置系统等核心功能。但存在数据存储方案薄弱、部分桌面特性缺失、搜索功能未实现等问题。

***

## 第一阶段：核心功能完善（高优先级）

### 1. 数据存储层重构 - SQLite 迁移

**问题：** 当前使用 LocalStorage，有 5MB 容量限制，不适合桌面应用
**方案：** 使用 Tauri SQL Plugin 迁移到 SQLite

**具体实现：**

* 添加 `@tauri-apps/plugin-sql` 依赖

* 重构 dbService.ts，实现 SQLite 数据库操作

* 数据迁移：首次启动时从 LocalStorage 迁移现有数据

* 数据库表设计：

  * `notes` 表 - 笔记数据

  * `tasks` 表 - 任务数据

  * `settings` 表 - 设置数据

  * `tags` 表 - 标签管理

  * `note_tags` 关联表 - 笔记-标签关系

### 2. 搜索功能实现

**问题：** Header 和 NoteList 有搜索框但无实际搜索功能
**方案：** 实现全文搜索，支持标题、内容、标签搜索

**具体实现：**

* 添加搜索索引

* 实现实时搜索（防抖 300ms）

* 搜索结果高亮显示

* 支持搜索历史记录

### 3. 系统托盘功能

**问题：** 设置中有"最小化到托盘"选项但无实际功能
**方案：** 使用 Tauri 系统托盘 API

**具体实现：**

* 添加系统托盘图标

* 实现托盘菜单（显示/隐藏、新建笔记、退出）

* 点击托盘图标显示/隐藏窗口

* 最小化到托盘功能

### 4. 全局快捷键

**问题：** 设计文档提到但未实现
**方案：** 使用 Tauri Global Shortcut API

**具体实现：**

* Ctrl+Shift+N: 新建笔记

* Ctrl+Shift+M: 显示/隐藏窗口

* Ctrl+Shift+F: 快速搜索

* 支持自定义快捷键

## 第二阶段：功能增强（中优先级）

### 5. 标签管理系统

**问题：** 笔记有标签字段但无标签管理界面
**方案：** 添加标签管理页面和功能

**具体实现：**

* 标签云/标签列表页面

* 创建/编辑/删除标签

* 按标签筛选笔记

* 标签颜色自定义

* 热门标签统计

### 6. 数据导入功能

**问题：** 设置页面有导入设置但无实际导入功能
**方案：** 实现多格式数据导入

**具体实现：**

* 支持导入 Markdown 文件（批量导入文件夹）

* 支持导入 JSON 数据

* 支持从其他笔记软件导入（如 Bear、Notion 导出格式）

* 导入预览和冲突处理

### 7. 编辑器增强

**问题：** 依赖中有 @codemirror 但未实际使用
**方案：** 集成 CodeMirror 6 编辑器

**具体实现：**

* Markdown 语法高亮

* 自动补全（标题、列表、代码块）

* Vim/Emacs 模式支持（可选）

* 图片粘贴上传

* 表格编辑器

### 8. PDF 导出功能

**问题：** 目前仅支持 md/txt/html/json 导出
**方案：** 添加 PDF 导出支持

**具体实现：**

* 使用 html2canvas + jsPDF 或 puppeteer

* 支持导出样式模板

* 页眉页脚自定义

## 技术要点

### Tauri 插件使用

```toml
# Cargo.toml
[dependencies]
tauri-plugin-sql = { version = "2", features = ["sqlite"] }
tauri-plugin-global-shortcut = "2"
tauri-plugin-notification = "2"
```

### 数据库迁移策略

1. 应用启动时检查数据库是否存在
2. 不存在则创建表结构
3. 检查 LocalStorage 是否有旧数据
4. 有则迁移到 SQLite，然后清空 LocalStorage
5. 添加版本标记，避免重复迁移

### 搜索实现方案

* 使用 SQLite FTS (Full-Text Search) 扩展

* 或内存中构建搜索索引（fuse.js）

* 支持中文分词（使用 jieba 或类似库）
